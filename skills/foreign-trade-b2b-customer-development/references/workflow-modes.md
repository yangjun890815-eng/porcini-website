# 开发方式

## 入口引导

用户只启动 Skill 但没有给任务时，用这个引导：

```text
你可以从以下方式开始：
1. 发一个公司官网，我分析单个客户。
2. 发产品名称 + 目标国家，我按产品开发客户。
3. 发一组关键词，我扩展关键词并开发潜在客户。
4. 发行业 + 地区，我按行业开发客户。
5. 发国家/地区 + 客户类型，我按市场开发客户。
6. 上传 Excel/CSV，我补全已有客户名单。
7. 发竞品品牌或同行网站，我找相似渠道和潜在买家。
8. 发展会名单/协会名单/目录名单，我帮你筛选和整理。
```


## 运行模式

执行前先判断：

```text
自动开发模式：当前 Agent 有网页搜索、网页读取、表格/文件输出能力，可以从零开发客户。
半自动整理模式：缺少网页搜索，但用户提供公司名单、官网列表、搜索结果、展会名单、Excel/CSV、PDF或网页材料，可以筛选整理。
策略模式：没有网页搜索和网页读取，也没有用户提供名单或材料，不能生成真实客户名单，只能输出搜索词、Google搜索式和开发路径。
```

从零开发客户最低需要：网页搜索 + 网页读取 + 表格/文件输出。

## 按产品开发

适合用户给出产品名、材料、用途、关键词。

步骤：

1. 把产品扩展成英文关键词、行业词、应用场景词。
2. 找进口商、经销商、批发商、品牌商、零售商、项目商。
3. 验证官网是否真的有相关品类。
4. 输出一个 `客户开发总表`。

## 按国家/地区开发

适合用户说“开发德国客户”“开发中东市场”。

步骤：

1. 确定国家和语言。
2. 加入当地语言关键词。
3. 找本地目录、协会、展会、经销商名单。
4. 按官网确认和品类验证筛选。

## 按行业开发

适合用户给出行业，例如建材、宠物用品、汽配、家居、机械、包装。

步骤：

1. 拆行业上下游。
2. 找不同客户类型。
3. 区分终端客户、渠道商、品牌商、项目商。
4. 输出开发建议。

## 按竞品开发

适合用户给出竞品品牌或竞争对手。

步骤：

1. 找竞品经销商、零售商、合作商、展会同场公司。
2. 确认这些公司是否经营类似产品。
3. 判断是否适合作为替代供应商切入。

## 按展会/协会/目录开发

适合用户给出展会名单、协会名单、目录名单或采购商列表。

步骤：

1. 先清洗名单，去掉媒体、个人、重复公司和明显不相关对象。
2. 给每家公司补官网。
3. 做官网确认、品类验证、客户类型判断。
4. 合并进一个 `客户开发总表`。

## 补全已有名单

适合用户上传 Excel/CSV 或粘贴公司名。

步骤：

1. 保留用户原始字段。
2. 补官网、品类、客户类型、联系方式。
3. 标注缺失项和需要人工确认项。
4. 不拆成多个总表，最终合并成一个 `客户开发总表`。
