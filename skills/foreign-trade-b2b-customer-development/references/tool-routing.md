# 工具能力路由

目标：让 Agent 尽量调用真实可用工具完成取证，减少模型凭印象输出。

## 支持环境

- Windows
- macOS
- WSL
- Codex
- Hermes
- OpenClaw 或其他支持 Skill 的 Agent

## 核心原则

1. 按“工具能力”路由，不绑定固定工具名称。
2. 有工具时，Agent 必须自己调用工具完成对应步骤。
3. 没工具或页面受限时，不得装作查过；要让用户提供资料或标注未查询原因。
4. 只检查公开可访问的信息。
5. 不登录、不读取 Cookie、不绕过验证码、登录墙、付费墙、反爬或平台风控。
6. 工具结果也要做业务判断，不盲信摘要。
7. 每次输出要告诉用户：查了什么、没查什么、哪里需要人工确认。

## 开始前能力核查

正式开发前，先根据当前 Agent 暴露的工具判断这些能力：

```text
网页搜索
网页读取
浏览器/动态页面
文档/PDF读取
表格/文件处理
公司信息辅助
域名/官网判断
邮箱验证
社媒公开页读取
地图公开信息查询
```

状态写：

```text
可用
不可用
未显示
本轮未用
```


## 运行模式和最低工具要求

正式开发前，先判断本轮模式：

```text
自动开发模式：有 网页搜索 + 网页读取 + 表格/文件输出，可以从零开发客户。
半自动整理模式：缺少网页搜索，但用户提供公司名单、官网、搜索结果、展会名单、Excel/CSV、PDF或网页材料，可以筛选整理。
策略模式：没有网页搜索，也没有网页读取，用户也没有提供名单或网页材料，只能输出搜索策略，不能生成真实客户名单。
```

最低工具要求：

- 从零开发客户最低需要 `网页搜索 + 网页读取 + 表格/文件输出`。
- 缺少网页搜索，不能从零发现客户。
- 缺少网页读取，不能做官网确认和品类验证。
- 缺少表格/文件输出，可以先输出 Markdown/CSV 文本，并说明文件输出受限。

## 能力映射

| 阶段 | 工具能力 | Codex 常见工具 | Hermes/其他 Agent 处理方式 | 不可用时 |
|---|---|---|---|---|
| 候选发现 | 搜索/网页搜索 | web search、搜索 MCP | 使用内置搜索工具 | 让用户提供搜索结果/名单；只给搜索式 |
| 官网确认 | 网页读取 | fetch、browser/web 工具 | 使用网页读取工具 | 让用户提供官网文字/截图；不得写官网已确认 |
| 动态页面 | 浏览器/页面自动化 | Playwright、Browser Use | 只打开已知公开页面 | 标注打不开/需要人工确认 |
| 品类验证 | 网页/PDF读取 | fetch、docalyze、文件工具 | 读取产品页、目录、案例 | 不得写品类已确认 |
| 联系方式 | 网页读取 | fetch/browser | 读取 Contact/Impressum/Dealer 页 | 联系方式状态写未找到/未查询 |
| 读表格 | 表格/文件工具 | tablebridge、Python/openpyxl | 使用表格工具 | 输出 CSV/Markdown |
| 公司背景 | 公司信息工具 | companyscope 或公开网页 | 只做辅助 | 以官网/用户资料为准 |
| 官网身份 | 域名/官网判断 | domainintel 或公开 WHOIS/页面信息 | 辅助判断域名和公司是否对应 | 标注需要人工确认 |
| 邮箱检查 | 邮箱验证 | email-verify | 只验证公开邮箱 | 不验证，标注未验证 |
| 社媒公开页 | 浏览器/网页读取/平台工具 | browser、公开页读取工具 | 每个客户都要尝试，只读公开可访问内容 | 停止该来源并记录原因 |
| Google Maps/公开地图 | 搜索/浏览器/网页读取 | web search、browser、地图公开页 | 每个客户都要尝试，核对公开地址/电话/营业状态 | 停止该来源并记录原因 |

## 门禁规则

- 没有搜索工具，也没有用户给的候选名单：不得输出真实客户名单，只能输出搜索策略。
- 没有官网读取结果或用户提供官网材料：不得写 `官网已确认`。
- 没有产品页、分类页、目录、案例或用户提供材料：不得写 `品类已确认`。
- 没有联系页或公开联系方式来源：不得写“已找到联系方式”。
- 每个进入主表的客户都必须尝试社媒公开页和 Google Maps/公开地图查询；查不到、工具不可用或页面受限时，也要在 `客户开发总表` 写明状态和原因。
- 搜索结果摘要只能进入 `候选发现`，不能单独作为 `官网确认` 或 `品类验证`。
- 模型记忆只能用于关键词扩展，不能作为客户事实依据。

## 本轮查询说明

每轮结果建议增加 `本轮查询说明` 表或章节：

```text
阶段
需要的查询能力
实际使用方式
查过的来源
未查询的来源
未查询原因
对结果的影响
下一步补齐方式
```

常见写法：

```text
正常查询
用户提供资料
工具不可用
页面受限
未找到
需要人工确认
```

## 降级方式

如果没有对应工具，先判断运行模式，不要一上来让用户补一堆材料：

```text
自动开发模式：继续开发。
半自动整理模式：说明会基于用户提供材料整理。
策略模式：说明当前环境不能自动开发真实客户，只能输出搜索策略和需要补充的资料。
```

需要补充资料时，优先给出 1-3 种最省事的选择，例如：公司官网、搜索结果、网页内容、截图、PDF文字、Excel/CSV 或客户名单。不要把未查询内容写成已查询。

## WSL 注意

WSL 路径常见为：

```text
/home/用户名/...
/mnt/c/Users/用户名/...
```

Codex 在 WSL 中优先安装到：

```text
~/.codex/skills/foreign-trade-b2b-customer-development/
```

Hermes/通用 Agent 可使用：

```text
~/.agents/skills/foreign-trade-b2b-customer-development/
```

写文件时优先使用用户给的路径。不要硬编码 Windows 或 macOS 路径。
